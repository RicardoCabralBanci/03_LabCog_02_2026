# Skills - O que são e como criar

> Fonte: `skills/skill-creator/SKILL.md` do repositório MoltBot

## O que são Skills?

Skills são **pacotes modulares e autocontidos** que estendem as capacidades do agente, fornecendo conhecimento especializado, workflows e ferramentas. Funcionam como "guias de onboarding" para domínios ou tarefas específicas — transformam o agente de propósito geral em um agente especializado.

## O que uma Skill fornece

1. **Workflows especializados** — Procedimentos multi-etapa para domínios específicos
2. **Integrações com ferramentas** — Instruções para formatos de arquivo ou APIs
3. **Expertise de domínio** — Conhecimento específico da empresa, schemas, lógica de negócio
4. **Recursos empacotados** — Scripts, referências e assets para tarefas complexas/repetitivas

## Anatomia de uma Skill

```
skill-name/
├── SKILL.md (obrigatório)
│   ├── YAML frontmatter (name + description)
│   └── Instruções em Markdown
└── Recursos opcionais/
    ├── scripts/       → Código executável (Python/Bash)
    ├── references/    → Documentação carregada sob demanda
    └── assets/        → Arquivos usados na saída (templates, ícones)
```

### SKILL.md

- **Frontmatter YAML**: Campos `name` e `description` — é o que o agente lê para decidir QUANDO usar a skill
- **Body Markdown**: Instruções de uso — só carregado DEPOIS que a skill é acionada

### Scripts (`scripts/`)
- Para código que seria reescrito repetidamente
- Token-eficiente, determinístico, pode ser executado sem carregar no contexto

### References (`references/`)
- Documentação que o agente consulta sob demanda
- Mantém o SKILL.md enxuto, carregado só quando necessário

### Assets (`assets/`)
- Arquivos usados na saída (templates, imagens, logos)
- Não são carregados no contexto — usados diretamente no output

## Princípios-chave

### Concisão
A janela de contexto é um bem público. Só adicionar informação que o agente NÃO sabe por conta própria.

### Graus de liberdade
- **Alta liberdade**: Múltiplas abordagens válidas → instruções em texto
- **Média liberdade**: Padrão preferido com variação → pseudocódigo/scripts com parâmetros
- **Baixa liberdade**: Operações frágeis → scripts específicos, poucos parâmetros

### Revelação progressiva (3 níveis)
1. **Metadata** (name + description) — sempre no contexto (~100 palavras)
2. **Body do SKILL.md** — quando a skill é acionada (<5k palavras)
3. **Recursos** — sob demanda (ilimitado, pois scripts rodam sem carregar no contexto)

## Processo de criação (6 passos)

1. **Entender** a skill com exemplos concretos
2. **Planejar** recursos reutilizáveis (scripts, references, assets)
3. **Inicializar** com `init_skill.py`
4. **Editar** implementar recursos e escrever SKILL.md
5. **Empacotar** com `package_skill.py` → gera arquivo `.skill`
6. **Iterar** baseado em uso real

## Relevância para o LabCog

As Skills do MoltBot são um modelo interessante para criar módulos de conhecimento reutilizáveis. A estrutura SKILL.md + scripts + references + assets pode inspirar como organizamos nossos próprios módulos de automação.
