# Pesquisa no Repositório MoltBot - Índice

## Objetivo
Investigar como o MoltBot utiliza Skills, Hooks, Plugins e o sistema de Agentes para entender mecanismos de extensão e automação aplicáveis ao LabCog.

## Arquivos desta pesquisa

| #   | Arquivo                                  | O que cobre                                                                  |
| --- | ---------------------------------------- | ---------------------------------------------------------------------------- |
| 01  | Skills - O que são e como criar          | Pacotes modulares de conhecimento (SKILL.md + scripts + references + assets) |
| 02  | Hooks - Automação baseada em eventos     | Scripts event-driven para automatizar ações (command, agent, gateway events) |
| 03  | Plugins - Extensões modulares do sistema | Módulos TypeScript que registram ferramentas, comandos, serviços             |
| 04  | Agente - Runtime, Workspace e Bootstrap  | Como o agente opera: workspace, arquivos de bootstrap, sessões               |

## Resumo comparativo

```
Complexidade:    Skills (baixa) → Hooks (média) → Plugins (alta)
Momento:         Skills (sob demanda) → Hooks (evento) → Plugins (sempre carregado)
Propósito:       Skills (conhecimento) → Hooks (automação) → Plugins (funcionalidade)
```

## Insights para o LabCog

1. **Skills** → Modelo para criar módulos de conhecimento por persona/projeto
2. **Hooks** → Automação de sessão (salvar estado, log, preparar contexto)
3. **Plugins** → Ferramentas customizadas e integrações
4. **Bootstrap files** → Base para configurar ambientes por persona (SOUL.md, AGENTS.md, etc.)
