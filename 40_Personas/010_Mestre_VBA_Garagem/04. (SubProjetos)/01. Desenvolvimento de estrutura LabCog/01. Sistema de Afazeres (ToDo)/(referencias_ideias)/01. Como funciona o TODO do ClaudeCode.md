# Como funciona o TODO do Claude Code

O Claude Code possui um sistema de tarefas integrado em memória, disponível durante a conversa.

## Ferramentas

| Ferramenta     | Função                                                  |
|----------------|---------------------------------------------------------|
| **TaskCreate** | Cria tarefa com título, descrição e status `pending`    |
| **TaskList**   | Lista todas as tarefas com ID, status e dependências    |
| **TaskGet**    | Retorna detalhes completos de uma tarefa por ID         |
| **TaskUpdate** | Altera status, descrição, dono ou dependências          |

## Ciclo de vida de uma tarefa

```
pending  →  in_progress  →  completed
                              │
                              └→ deleted (remoção permanente)
```

## Campos de uma tarefa

- **subject**: Título curto e imperativo (ex: "Criar banco de dados")
- **description**: Detalhamento completo do que precisa ser feito
- **activeForm**: Texto no gerúndio exibido enquanto a tarefa está ativa (ex: "Criando banco de dados")
- **status**: `pending`, `in_progress`, `completed` ou `deleted`
- **owner**: Agente responsável
- **metadata**: Dados extras em formato chave-valor

## Dependências entre tarefas

- **blocks**: Lista de tarefas que NÃO podem começar até esta ser concluída
- **blockedBy**: Lista de tarefas que PRECISAM ser concluídas antes desta começar

Exemplo: "Ler arquivos legados" **blocks** → "Esboçar nova estrutura"

## Limitações

- **Volátil**: As tarefas existem APENAS durante a conversa. Ao encerrar, tudo se perde.
- **Sem persistência**: Não há banco de dados ou arquivo — vive apenas em memória.
- **Sem histórico**: Não guarda registro de tarefas de conversas anteriores.

## Conclusão

É útil para organizar o trabalho **dentro de uma sessão**, mas precisa de uma solução persistente para acompanhamento a longo prazo — o que motiva o desenvolvimento do nosso próprio sistema.
