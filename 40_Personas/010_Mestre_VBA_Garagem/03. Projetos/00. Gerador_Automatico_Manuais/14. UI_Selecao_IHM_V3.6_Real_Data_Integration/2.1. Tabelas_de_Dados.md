### Notas para a Estrutura de Dados

1. **Rigor Visual**: IHMs que não possuem todos os capítulos (ex: ID 6) mostrarão apenas os mapeados. Se o objetivo é que o usuário SEMPRE veja os 4 espaços, o código de Fallback (v1.3 do 70.3.1) garantirá a exibição dos itens vazios como "Bloqueados".
2. **Path Relativo**: Caminhos devem seguir o padrão `templates/...`.
