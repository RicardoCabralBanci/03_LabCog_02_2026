## 2. Estrutura de Dados em Memória (VBA Types)

O código VBA deve ser mapeado exatamente para os nomes das colunas acima.

```vba
' Representa um CARD na Zona A
Public Type T_IHM_Card
    ID As Long              ' Coluna ID (1, 2, 3...)
    Fabricante As String    ' Coluna Fabricante (Filtro e Cor do Badge)
    Modelo As String        ' Coluna Modelo (Título do Card)
    Tamanho As String       ' Coluna Tamanho (Badge 1)
    Engine As String        ' Coluna Engine (Badge 2)
    PathThumb As String     ' Coluna Path_Thumbnail
End Type
```

---

## 3. Lógica de Sincronização (Mapeamento)

Ao carregar os dados da planilha para o Array no VBA:
1.  **Fabricante**: Determina em qual grupo da Sidebar o item aparece e a cor do Badge.
2.  **Modelo**: É o texto principal de identificação.
3.  **Tamanho e Engine**: São transformados nos "Badges" visuais na parte inferior do card.
4.  **ID**: É o "RG" do item. Quando você clica no card, o VBA envia esse número para a Zona B buscar os manuais correspondentes.
