## 1. Visão Geral
Este módulo não contém lógica de processamento de dados. Ele é puramente um **Controlador de Navegação de Interface**. Ele mapeia botões da interface (provavelmente botões de "Próxima Página" ou "Menu" dentro do Excel) para a ativação de abas específicas.

---

## 2. Dissecção

### 2.1. O Padrão de Ativação
O módulo consiste em dezenas de funções quase idênticas:
```vba
Sub BTRPG1(Optional byDummy As Byte)
    Planilha5.Activate
End Sub
```
*   **Significado**: `BTR` (Máquina) + `PG1` (Página 1).
*   **Destino**: Codename `Planilha5`.

### 2.2. Mapeamento de Tecnologias
O script revela a organização interna do Workbook:
*   **BTR**: Planilhas 5 a 11.
*   **GTR**: Planilhas 13 a 19.
*   **DVD**: Planilhas 21 a 25.
*   **PET**: Planilhas 26 a 27.
*   **CIP**: Planilhas 29 a 32.
*   **CMX**: Planilhas 35 a 36.
*   **CCMX**: Planilhas 38 a 40.

---

## 3. Diagnóstico (O Cirurgião)

### 3.1. Inutilidade Sistêmica
Este código existe apenas para suportar uma interface de usuário baseada em botões dentro das planilhas. Em um ambiente de automação real, o usuário não precisa "navegar" entre abas; o script lê todas elas em segundo plano.

### 3.2. Manutenibilidade Zero
O uso de `PlanilhaX` (Codenames) é ligeiramente melhor que o nome da aba (que o usuário pode mudar), mas ainda assim é frágil. Se uma planilha for deletada e recriada, o Codename muda e o script morre.

---

## 4. Veredito de Migração: **ELIMINAR COMPLETAMENTE**

Este módulo **não deve ser migrado** para Python.
1.  **Sem Interface Excel**: A nova ferramenta não terá botões de navegação dentro do Excel.
2.  **Acesso Direto**: No Python, acessaremos as abas pelo nome (`df = pd.read_excel(file, sheet_name='GTR Áreas e segmentos')`).
3.  **Lógica Centralizada**: A ordem das abas será definida por um arquivo de configuração ou pela própria sequência do manual, eliminando a necessidade de macros de "próximo/anterior".
