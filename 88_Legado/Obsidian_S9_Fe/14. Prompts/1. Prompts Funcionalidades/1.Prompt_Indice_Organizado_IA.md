# Prompt para IA: Geração de Índice Mestre Organizado Tematicamente

**Objetivo:** Analisar a lista de títulos de notas fornecida e gerar um índice organizado tematicamente em formato Markdown, inspirado em métodos como Zettelkasten (agrupamento por ideias) ou PARA (áreas/projetos), mas adaptado aos temas que emergem dos próprios títulos.

**Contexto:**

- **Entrada:** Uma lista de caminhos relativos de arquivos `.md` representando as notas em um vault Obsidian.
    
- **Saída Desejada:** Um arquivo Markdown (`1.2 Indice Mestre Organizado pela IA.md`) que agrupa os links para essas notas sob cabeçalhos temáticos relevantes.
    

**Instruções Detalhadas:**

1. **Analise os Títulos:** Leia cuidadosamente todos os títulos de notas fornecidos.
    
2. **Identifique Temas/Clusters:** Procure por padrões, palavras-chave recorrentes e conceitos relacionados para identificar os principais temas ou agrupamentos lógicos presentes no vault (ex: "Mente e Cognição", "Projetos de Escrita", "Referências de IA", "Finanças", "Ideias Aleatórias", etc.). Use sua capacidade de compreensão semântica. **Instrução Específica:** Identifique notas cujos títulos sugiram listas de tarefas, itens a comprar, ou informações de referência rápida e agrupe-as sob o cabeçalho `## Utilidades / Dia a Dia`.
    
3. **Crie Cabeçalhos Temáticos:** Para cada tema principal identificado (incluindo "Utilidades / Dia a Dia"), crie um cabeçalho Markdown (nível 2: `##`).
    
4. **Liste Notas Relacionadas:** Sob cada cabeçalho temático, liste as notas (usando o formato de link `[[Caminho/Relativo/Nota.md]]`) cujos títulos se encaixam naquele tema. Use marcadores (`-` ou `*`).
    
5. **Notas "Órfãs" (Opcional):** Se algumas notas não se encaixarem claramente em nenhum tema principal (incluindo "Utilidades / Dia a Dia"), você pode agrupá-las sob um cabeçalho genérico como `## Outros Tópicos` ou `## Notas Diversas`.
    
6. **Estrutura e Clareza:** Organize a saída de forma lógica e fácil de ler. Use sub-cabeçalhos (`###`) se fizer sentido para subtemas dentro de um tema principal.
    
7. **Formato:** Gere a saída **apenas** em Markdown válido. **NÃO** inclua cercas de bloco de código (```) no início ou fim da sua resposta.
    
8. **Conciso:** Mantenha as descrições (se houver) breves. O foco são os agrupamentos e os links.
    

**Exemplo de Saída Esperada:**

```
# Índice Mestre Organizado por IA

*Gerado em: [Data]*

## Utilidades / Dia a Dia

- [[Lista de Compras Supermercado.md]]
- [[Tarefas Pendentes Casa.md]]
- [[Contatos Importantes.md]]

## Tema Principal A

- [[PastaA/NotaRelacionada1.md]]
- [[PastaA/SubPasta/NotaRelacionada2.md]]
- [[NotaRaizTemaA.md]]

## Tema Principal B

### Subtema B1

- [[PastaB/NotaSubtemaB1.md]]
- [[PastaB/OutraNotaB1.md]]

### Subtema B2

- [[PastaB/NotaSubtemaB2.md]]

## Outros Tópicos

- [[IdeiaSolta1.md]]
- [[ReferenciaRapida.md]]
```

**Foco:** O objetivo é criar uma visão geral temática útil do vault, agrupando notas relacionadas com base em seus títulos. A qualidade do agrupamento é mais importante que a perfeição absoluta.
