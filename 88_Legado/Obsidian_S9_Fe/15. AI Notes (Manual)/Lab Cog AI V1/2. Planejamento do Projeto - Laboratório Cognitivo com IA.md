# Planejamento do Projeto: Laboratório Cognitivo com IA

Este documento detalha o planejamento para a criação e desenvolvimento do projeto "Laboratório Cognitivo com IA", focado inicialmente no ambiente Obsidian.

## 1. Visão e Objetivos do Projeto

- **Visão:** Criar um sistema de gestão de conhecimento pessoal (PKM) inteligente e otimizado, utilizando o **Obsidian como base**, mas turbinado com capacidades de IA para automatizar e aprimorar o fluxo de captura, organização, sumarização, indexação e conexão de **notas**, utilizando prompts customizáveis para tarefas específicas.
    
- **Objetivos do Projeto (em ordem de desenvolvimento/prioridade):**
    
    1. **Leitura e Processamento:** Capacitar o sistema (via script Python) a ler e processar dados das notas `.md` existentes no vault Obsidian, incluindo metadados e conteúdo, servindo como base para as demais funcionalidades. _(Status: Parcialmente alcançado com scripts de leitura)_.
        
    2. **Geração de Artefatos:** Implementar a capacidade de gerar automaticamente novos arquivos `.md` no vault (ex: Notas de Resumo em `13. AI Notes`, Arquivo Índice Mestre) com base na análise das notas existentes via IA. _(Status: Nota de Resumo parcialmente implementada)_.
        
    3. **Criação e Manutenção do Índice Mestre:** Desenvolver a funcionalidade específica para gerar e/ou atualizar um "Arquivo Índice Mestre" (localizado em `13. AI Notes` e nomeado como `1. Nome_Indice.md`).
        
        - Gerar lista inicial de links formatados (`[[...]]`) para todas as notas elegíveis do vault.
            
        - **Ordenar a lista de links (ex: por data de criação das notas correspondentes, alfabeticamente).** _(Preferência: Data de Criação)_.
            
        - Implementar lógica para identificar notas novas/removidas desde a última geração/atualização do índice.
            
        - **(Avançado/Opcional)** Implementar lógica para detectar notas modificadas (usando **Timestamps, Hashes de Conteúdo ou integração Git**) e potencialmente marcá-las no índice ou permitir re-processamento direcionado. (Atividade 2.9 - _(Status: Pendente)_).
            
    4. **Sistema de Prompts Específicos:** Implementar um sistema onde prompts armazenados em uma pasta dedicada (ex: `14. Prompts`) possam guiar o comportamento da IA para tarefas ou arquivos específicos (como a geração/atualização do Arquivo Índice Mestre ou outras análises futuras). O script deverá ser capaz de localizar e utilizar o prompt relevante para a tarefa em execução. (Atividade 2.10 - _(Status: Pendente)_).
        
    5. **Análise e Conexão via IA:** Implementar funcionalidades de IA para análise semântica e conexão entre notas:
        
        - Sugestão e/ou criação automática de links (`[[...]]`) entre notas relevantes, utilizando prompts customizáveis (seja a nota de orientação geral ou prompts específicos da pasta 14). (Atividade 2.3 - _(Status: Pendente)_).
            
        - Assistência na criação/modificação de conteúdo de notas (ex: sumarização de trechos, expansão de ideias). (Atividade 2.2 - _(Status: Pendente)_).
            
        - Geração de "Notas Pai" que sumarizem grupos específicos de notas (requer definição de critérios). (Atividade 2.8 - _(Status: Pendente)_).
            
    6. **Implementação de OCR (Opcional/Secundário):** Integrar uma solução de OCR para transcrever notas manuscritas (Excalidraw, imagens) para texto digital dentro do Obsidian. (Atividade 2.4 - _(Status: Pendente)_).
        
    7. **Integração e Avaliação:** Garantir integração fluida das funcionalidades com o fluxo de trabalho do Obsidian e avaliar a performance, usabilidade e precisão das ferramentas implementadas. _(Status: Contínuo)_.

## 3. Escopo do Projeto (Foco na Visão Original, OCR Secundário) _(Antiga Seção 2)_

- **Entregáveis Principais:**
    
    - Um conjunto de **scripts Python funcionais** que implementem as funcionalidades principais de IA (assistência de notas, linkagem, organização, sumarização/Notas Pai, **geração/manutenção de Índice Mestre**, **uso de prompts específicos**) operando sobre o **vault Obsidian**.
        
    - Um script Python funcional para OCR de notas manuscritas (Entregável Secundário / Opcional).
        
    - Documentação básica explicando como instalar, configurar e executar os scripts.
        
    - (Opcional) Um sistema de orçamento gerenciado dentro do próprio vault Obsidian.
        
- **Fora do Escopo (Confirmado):**
    
    - Indexação e organização de arquivos fora do vault Obsidian especificado.
        
    - Criação de um aplicativo de notas completamente novo e independente.
        
    - Desenvolvimento de um plugin Obsidian nativo.
        
    - Funcionalidades de IA que vão além do processamento de texto e notas (exceto OCR, se implementado).
        
    - Garantia de funcionamento perfeito com _todos_ os temas e plugins de terceiros.
        
    - Interface gráfica polida para os scripts.
        

## 4. Recursos Necessários

- **Equipe:** O usuário e eu (Gemini).
    
- **Tecnologias:**
    
    - Base: Obsidian (arquivos `.md`).
        
    - Core: Python 3.x, bibliotecas (`watchdog`, `python-markdown`, requests, SDKs de APIs IA).
        
    - IA: API Gemini (ou outra a definir).
        
    - OCR (Secundário/Opcional): Tesseract (local) ou API Google Vision (ou outra a definir).
        
    - Automação (Opcional): n8n, agendador de tarefas do SO.
        
    - Backend/Dados (Opcional): Firebase (a avaliar).
        
    - Pesquisa/Auxiliar (Opcional): NotebookLM.
        
- **Dados:** Acesso (indireto, via scripts) ao vault Obsidian do usuário para testes. _(Amostras de notas manuscritas necessárias apenas se/quando o OCR for desenvolvido)_.
    
- **Orçamento (se aplicável):** A ser detalhado. Custos de APIs (IA). _(Custos de API OCR dependentes da implementação secundária)_. **Gerenciamento:** No vault Obsidian.
    

## 5. Premissas

- O usuário possui um vault Obsidian funcional.
    
- O usuário utiliza o plugin Excalidraw (relevante principalmente para OCR secundário).
    
- O usuário tem capacidade/ambiente para executar scripts Python localmente.
    
- Acesso à internet para APIs externas (IA).
    
- APIs/ferramentas escolhidas têm performance/precisão aceitáveis.
    
- Scripts Python terão permissão para ler/escrever no diretório do vault Obsidian.
    
- Tecnologias listadas são viáveis.
    
- O escopo do projeto limita-se ao diretório do vault Obsidian especificado (conforme Seção 3). _(Referência atualizada)_
    

## 6. Riscos e Mitigações

|---|---|---|---|

## 7. Critérios de Sucesso

- Funcionalidades principais de IA (assistência, linkagem, sumarização, **geração de índice ordenado**, **uso de prompts específicos**) via scripts Python operando com precisão > X% (a definir).
    
- OCR (Critério Secundário / Opcional).
    
- Tempo de execução dos scripts (principais) dentro de limites aceitáveis.
    
- **Arquivo Índice Mestre gerado corretamente, ordenado por data de criação e refletindo o estado atual (novos/removidos) das notas elegíveis.**
    
- **Sistema de prompts específicos funcionando conforme esperado para tarefas designadas.**
    
- Integração com o fluxo de trabalho do Obsidian percebida como útil.
    
- Entregáveis principais (scripts IA, documentação) concluídos.
    
- Sistema de orçamento no vault funcional (se implementado).
    

## 8. Visão Expandida / Ambição Futura (Escopo Fora do Projeto Atual)

- **Visão (Expandida):** Evoluir o conceito para um sistema de organização e inteligência que abranja **todos os arquivos do computador do usuário**, não apenas o vault Obsidian. Utilizar modelos de IA com grandes janelas de contexto (ex: Llama 3) para indexar, analisar, resumir, organizar e potentially criar links entre diversos tipos de arquivos (documentos, imagens, código, etc.) em todo o sistema de arquivos.
    
- **Desafios Associados à Visão Expandida:**
    
    - Escala, Diversidade de Arquivos, Indexação, Limites de Contexto, Performance, Relevância/Ruído, Segurança/Privacidade.
        
- **Status:** Esta visão expandida representa uma **mudança fundamental e de altíssima complexidade**. Deve ser considerada, no máximo, como uma **possível evolução futura e distante**. **Este escopo expandido NÃO faz parte do projeto atual.**
